# Project Report: EduSphere - A Role-Based Student Management System

## 1. Abstract

**EduSphere** is a robust, role-based student management system designed to streamline the academic and administrative processes of educational institutions. Built with a Python Flask backend and a dynamic frontend UI, it accommodates administration workflows, faculty oversight, and student interactions seamlessly. The system ensures robust data persistence using PostgreSQL for production and SQLite for local development. By integrating core features like attendance tracking, course management, performance evaluation, and AI-powered assistance, EduSphere aims to provide a centralized, efficient, and user-friendly platform for all institutional stakeholders.

## 2. Introduction

### 2.1 Purpose
The purpose of EduSphere is to digitize and centralize the daily operations of a college or school. It provides dedicated modules for administrators, faculty, and students, ensuring that each user has access to tools and information pertinent to their role.

### 2.2 Scope
The system covers the entire student lifecycle within an institution, including:
- User onboarding and role management.
- Course and lecture scheduling.
- Attendance and marks tracking.
- Report card generation and distribution.
- Institutional announcements and personalized notifications.
- Inter-role communication (Chat support).
- AI assistance for quick, academic guidance.

## 3. Technology Stack

- **Frontend:** HTML5, Vanilla CSS, JavaScript. Features an adaptive Glassmorphism UI with local preference syncing (Dark Mode).
- **Backend:** Python 3, Flask.
- **Database:** PostgreSQL (Production on Render) & SQLite (Local Development).
- **Authentication:** Role-encoded, Flask session-based authentication with CSRF protection.
- **AI Integration:** Google Gemini AI Model (gemini-2.5-flash) for intelligent tutoring and assistance.
- **Deployment:** Configured for automatic deployment on Render using Gunicorn as the WSGI HTTP Server.

## 4. System Architecture & Database Design

EduSphere is built on a relational database architecture. The system employs structured relationships to maintain data integrity across multiple interconnected entities:

- **Users Entity:** Centralized authentication table (`users`) tracking Admin, Faculty, and Students with hashed passwords.
- **Profiles:** Extended profile tables for `faculty_profiles` (department, bio) and `student_profiles` (course, year, mentor).
- **Academics:** Tables for `courses`, `course_lectures`, `subjects`, and `course_enrollments` link faculty and students to specific classes.
- **Evaluation:** `marks`, `attendance_summary`, and `attendance_sessions` securely store academic performance.
- **Communication:** `announcements`, `notifications`, and direct messaging tables (`support_messages`, `admin_faculty_messages`) manage intra-system chats.
- **Records:** `report_cards` and `report_card_entries` handle structured academic transcripts.

## 5. Modules & Key Features

### 5.1 Admin Module
The administrator is the superuser of EduSphere, possessing complete autonomy over the system:
- **User Management:** Create, update, and manage accounts for all students, faculty, and other admins.
- **Course & Subject Allocations:** Define courses, assign faculty to subjects, and manage institution-wide curricula.
- **Announcements:** Broadcast priority announcements to the entire institution.
- **Performance Exports:** Generate and export system-wide performance data.
- **Faculty Support Panel:** Dedicated chat link to communicate directly with faculty members.

### 5.2 Faculty Module
Faculty members have access to course-specific tools and mentorship panels:
- **Mentorship Tracking:** Seamless tracking of assigned (mentored) students.
- **Academic Management:** Submit dynamic attendance, record marks, and deploy assignments for their respective courses.
- **Schedule Management:** Timeline-managed course schedules and lecture status tracking.
- **Support Panels:** Direct communication channels with both the Administration and assigned students.

### 5.3 Student Module
Students have a personalized dashboard reflecting their academic progress:
- **Live Syncing:** Real-time updates on enrolled courses, progress evaluations, and timeline of lectures.
- **Report Cards:** Access and download semester report cards dynamically generated by the system.
- **AI Tutoring:** Built-in AI chat assistance tailored to provide safe, practical academic help and study guidance.
- **Mentor Conversations:** Direct, dynamic chat support with their assigned faculty mentor.

## 6. Security and Data Protection

EduSphere is designed with industry-standard security practices:
- **Authentication:** Passwords are securely hashed using Werkzeug (`generate_password_hash`, `check_password_hash`).
- **Rate Limiting:** Login attempts are heavily rate-limited and locked after successive failures to prevent brute-force attacks.
- **Session Security:** Cookies are HTTPOnly, secure, and mapped with strict SameSite policies.
- **CSRF & Header Protection:** The application enforces CSRF tokens for all state-changing API requests. Strict headers like `Content-Security-Policy`, `X-Frame-Options`, and `X-Content-Type-Options` protect against XSS and clickjacking.

## 7. Future Scope

- **Mobile Application:** Extending the platform to dedicated native iOS and Android applications.
- **Advanced Analytics:** Adding predictive analytics for student performance and drop-out risk assessment.
- **Payment Gateway Integration:** Handling fee submissions and transactions directly within the portal.

## 8. Conclusion

EduSphere successfully demonstrates the practical implementation of a full-stack educational management platform. By cleanly separating concerns across roles, ensuring strict security standards, and maintaining a responsive UI, the system proves to be a viable, modern solution for institutional administration and student engagement.
